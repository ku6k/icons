# Changelog - Color Tokens Dark

All notable changes to the Dark Mode Color Tokens module will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.0.0] - 2025-01-01

### Added
- Initial release of KU6K dark mode color tokens (future implementation)
- Dark theme companion to color-tokens@1.0.0
- Enhanced dark mode color palette with improved contrast
- CSS custom properties with --ku6k- prefix
- Compatibility aliases for existing token names
- Dark mode semantic colors (success, warning, error)

### Features
- 🌙 Complete dark mode color system
- 🎨 Enhanced primary colors for dark theme
- 🔧 CSS custom properties ready
- ♿ WCAG AA compliant contrast ratios in dark mode
- 🎯 Future-proof architecture for dark theme evolution

### Tokens Added
- `--ku6k-primary`: Enhanced primary color for dark mode
- `--ku6k-background`: Dark background colors
- `--ku6k-foreground`: Light text colors for dark backgrounds
- `--ku6k-card`: Dark surface colors
- `--ku6k-muted`: Dark muted colors
- `--ku6k-border`: Dark border colors
- `--ku6k-destructive`: Dark mode error colors

## Version Guidelines

### PATCH (2.0.X)
- Color refinements and contrast improvements
- Dark mode specific accessibility fixes
- Performance optimizations

### MINOR (2.X.0)
- New dark mode variants within existing families
- Additional dark mode semantic colors
- New dark theme utilities

### MAJOR (X.0.0)
- Breaking changes to dark theme structure
- Major dark mode architecture changes
- Incompatible changes with light mode tokens
